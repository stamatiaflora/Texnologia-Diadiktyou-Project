# 1. Σενάριο

Στο πλαίσιο της εργασίας επιλέξαμε να αναπτύξουμε ένα σύστημα διαχείρισης αποθήκης για κατάστημα γυναικείου αθλητικού ρουχισμού.

Η ανάγκη για μια τέτοια εφαρμογή προκύπτει από το γεγονός ότι ένα κατάστημα πρέπει να μπορεί να οργανώνει και να παρακολουθεί τα προϊόντα που διαθέτει στην αποθήκη του. Μέσω της εφαρμογής ο χρήστης μπορεί να καταχωρεί νέα προϊόντα, να τα αποθηκεύει στη βάση δεδομένων και να βλέπει όλα τα διαθέσιμα προϊόντα που υπάρχουν στην αποθήκη.

Η εφαρμογή διαχειρίζεται προϊόντα όπως κολάν, αθλητικά τοπ, ζακέτες και ολόσωμες φόρμες. Με αυτόν τον τρόπο επιτυγχάνεται μια απλή αλλά λειτουργική οργάνωση των προϊόντων του καταστήματος.

# 2. Αρχιτεκτονική της Εφαρμογής

Η εφαρμογή αναπτύχθηκε ως web εφαρμογή με χρήση του περιβάλλοντος XAMPP.

Ο χρήστης αλληλεπιδρά με την εφαρμογή μέσω browser. Τα αιτήματα αποστέλλονται στον Apache Server, ο οποίος εκτελεί τον κώδικα PHP. Στη συνέχεια ο κώδικας επικοινωνεί με τη βάση δεδομένων MySQL για την αποθήκευση ή την ανάκτηση των δεδομένων.

Η εφαρμογή αποτελείται από τέσσερα βασικά αρχεία:

index.php

Αποτελεί την αρχική σελίδα της εφαρμογής και παρέχει πρόσβαση στις διαθέσιμες λειτουργίες.

add_product.php

Περιλαμβάνει τη φόρμα καταχώρησης νέου προϊόντος. Ο χρήστης συμπληρώνει τα στοιχεία του προϊόντος και αυτά αποθηκεύονται στη βάση δεδομένων.

view_products.php

Εμφανίζει όλα τα προϊόντα που υπάρχουν στη βάση δεδομένων σε μορφή πίνακα.

db.php

Περιέχει τις ρυθμίσεις σύνδεσης με τη βάση δεδομένων και χρησιμοποιείται από τα υπόλοιπα αρχεία για την επικοινωνία με τη MySQL.

Η γενική λειτουργία της εφαρμογής μπορεί να περιγραφεί ως εξής:

Χρήστης → Browser → Apache Server → PHP → MySQL Database

# 3. Δομή της Βάσης Δεδομένων

Για την αποθήκευση των δεδομένων δημιουργήθηκε η βάση δεδομένων:

warehouse_store

Στη βάση δημιουργήθηκε ένας πίνακας με όνομα:

products

Ο πίνακας περιλαμβάνει τα παρακάτω πεδία:

Πεδίο	Περιγραφή
id	Μοναδικός κωδικός προϊόντος
product_name	Όνομα προϊόντος
category	Κατηγορία προϊόντος
size	Μέγεθος
color	Χρώμα
quantity	Ποσότητα
price	Τιμή

Το πεδίο id χρησιμοποιείται ως πρωτεύον κλειδί (Primary Key), ώστε κάθε προϊόν να αναγνωρίζεται μοναδικά μέσα στη βάση δεδομένων.

Η δημιουργία και διαχείριση της βάσης πραγματοποιήθηκε μέσω του phpMyAdmin που παρέχεται από το XAMPP.

# 5. Δικτυακή Αρχιτεκτονική

Για το δεύτερο μέρος της εργασίας χρησιμοποιήθηκε το λογισμικό Cisco Packet Tracer.

Η τοπολογία που σχεδιάστηκε είναι κυκλική (Ring Topology) και αποτελείται από:

5 Routers Cisco 2911

2 Switches Cisco 2960

2 Υπολογιστές (PCs)

Οι δύο υπολογιστές βρίσκονται σε διαφορετικά δίκτυα και τοποθετήθηκαν σε αντίθετα σημεία της τοπολογίας, σύμφωνα με τις απαιτήσεις της εκφώνησης.

Οι συνδέσεις του δικτύου είναι οι εξής:

PC1 → SW1 → R1
R1 ↔ R2
R2 ↔ R3
R3 ↔ R4
R4 ↔ R5
R5 ↔ SW2 → PC2
R1 ↔ R5

Επιπλέον δημιουργήθηκε σύνδεση μεταξύ των R1 και R5 για την ύπαρξη εναλλακτικής διαδρομής μέσω OSPF.

Για τη δρομολόγηση χρησιμοποιήθηκε το πρωτόκολλο OSPF (Open Shortest Path First), το οποίο επιτρέπει την αυτόματη ανταλλαγή πληροφοριών δρομολόγησης μεταξύ των routers.

Για την υλοποίηση του δικτύου χρησιμοποιήθηκαν τα παρακάτω υποδίκτυα:

WAN Δίκτυα

10.0.0.0/30

10.0.0.4/30

10.0.0.8/30

10.0.0.12/30

10.0.0.16/30

LAN Δίκτυα

192.168.1.0/24

192.168.2.0/24

Μετά την ολοκλήρωση των ρυθμίσεων πραγματοποιήθηκαν έλεγχοι με τις εντολές:

show ip interface brief
show ip ospf neighbor
show ip route

Επίσης πραγματοποιήθηκε επιτυχής επικοινωνία με ping μεταξύ του PC1 και του PC2, γεγονός που επιβεβαιώνει τη σωστή λειτουργία του δικτύου και του πρωτοκόλλου OSPF.
